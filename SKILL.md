# PDF-to-Ebook Skill

将英文 PDF 翻译为中文并生成具有设计感的中文电子书。默认输出采用接近 `guizang-ppt-skill` 的 Swiss editorial / 研究报告电子杂志风格：高反差封面、网格系统、章节幕封、精致图表卡片与适合 A4 打印的正文排版。

## 触发条件

- 用户上传英文 PDF 并要求翻译成中文电子书
- 用户说"把这个 PDF 做成中文电子书"、"翻译这份 PDF 并排版"
- 任何涉及 PDF → 中文精排电子书的需求

## 工作流（5 阶段）

### 阶段 1：PDF 文本提取（确保高成功率）

按以下优先级依次尝试，每次尝试后检查结果完整性：

**优先级 1 — `pdf` 工具（内置）**
```
用 pdf 工具逐页提取全部文字。记录哪些页提取成功、哪些失败。
```

**优先级 2 — `pdftotext`（poppler）**
```
如果内置 pdf 工具有页面失败：
1. brew install poppler（如果未安装，等待最多 120s）
2. pdftotext -layout input.pdf output.txt
3. 检查失败页面是否被提取
```

**优先级 3 — PyMuPDF（Python）**
```
如果 pdftotext 也不完整：
1. pip3 install PyMuPDF（等待最多 120s）
2. 用 Python 脚本逐页提取：
   import fitz
   doc = fitz.open(pdf_path)
   for page in doc:
       print(page.get_text())
3. 检查失败页面
```

**优先级 4 — 图片 OCR（针对图片化页面）**
```
如果仍有页面是图片格式无法提取文字：
1. pip3 install PyMuPDF（如未安装）
2. 将失败页面导出为 PNG（用 PyMuPDF 的 page.get_pixmap()）
3. 用 autoglm-image-recognition skill 逐张 OCR
4. 合并 OCR 结果
```

**兜底方案 — 请求用户协助**
```
如果以上全部失败或某个方法超时（>120s）：
→ 告诉用户哪些页面无法提取
→ 请用户用 Preview 打开 PDF → 文件 → 导出为文本
→ 或用截图工具截取失败页面发给 Agent OCR
```

**⚠️ 关键原则：** 每个方法尝试不超过 1 次，安装命令超时 120s 即跳过。不在同一个方法上反复重试。优先用 `pdf` 工具（最快），逐级降级。

### 阶段 2：图表与图片审查（必须执行）

在翻译前必须判断 PDF 中是否存在需要保留的视觉信息，不能只提取正文文本。

1. **建立图表清单**
   - 在 `original.md` 或抽取文本中搜索 `Figure`、`Fig.`、`Table`、`Chart`、`Diagram`、`Appendix`、`图`、`表` 等标记。
   - 记录每个图/表的编号、标题、页码、正文引用位置。
   - 如果搜索不到明确标记，也要抽查页面渲染图，判断是否有未被文本抽取捕捉的图片、流程图、截图、公式图、雷达图、柱状图、统计表等。

2. **判断处理方式**
   - 对复杂图形、图表截图、雷达图、折线图、柱状图、流程图、模型架构图：优先从 PDF 页面渲染或裁剪为图片，保存到 `assets/`。
   - 对数据表格：优先重建为 Markdown 表格；如果表格太宽、版式复杂、抽取后容易错位，则同时保留原始裁剪图。
   - 对案例表、附录表：保留原始句子、分数、符号和关键列，必要时附中文说明。
   - 对装饰性图片、页眉页脚、机构 logo：通常不加入电子书，除非用户要求完整复刻。

3. **提取方法优先级**
   - 优先使用现有 PDF/图像工具或 Python PDF 库检查页面资源与页码。
   - 如果可以渲染页面，用页面 PNG 裁剪出图表区域，并保存为语义化文件名，例如 `assets/figure1_comet.png`、`assets/table2_pos_stats.png`。
   - 如果不能直接裁剪，使用 PDF 页面截图或用户提供的截图作为图表源。
   - 如果图表图片中包含重要文字且不可读，需要用中文图注补充说明。

4. **译文中的插入规则**
   - `translated.md` 必须在对应章节或“图表译注”章节加入 Markdown 图片引用：
     ```
     ![图1：中文图题](assets/figure1_xxx.png)
     ```
   - 图片下方必须加入简短中文图注，说明图/表展示什么、关键结论是什么。
   - 图表编号应与原文一致，例如 Figure 1 → 图 1，Table 3 → 表 3。
   - 最终 HTML/PDF 必须包含这些图表；交付前检查 `assets/` 中的图片是否被引用。

5. **完整性校验**
   - 交付前列出已处理的 Figure/Table 清单。
   - 对照原文图表编号，确认没有漏掉正文图、主要表格、附录表格。
   - 如果某个图表无法提取，明确告知用户编号、页码和原因。

### 阶段 3：翻译

1. 将提取的英文文本完整保存为 `original.md`
2. 逐章翻译为中文，保存为 `translated.md`
3. 翻译要求：
   - 术语保留英文原文（如 MVP、PMF、CAC、LTV、GTM）
   - 保持原文的强调（加粗）、列表、引用结构
   - 表格完整翻译
   - 章节标题格式：`# 第X章 中文标题`
   - 附录标题：`# 附录：中文标题`
4. 如果原文内容超过 20 页，分批翻译，每批保存追加

### 阶段 4：排版生成

使用项目内置的 `build_pdf.py` 脚本生成 HTML 和 PDF：

```
python3 build_pdf.py translated.md ebook.html
```

脚本会自动：
- 解析 Markdown 的章节结构
- 渲染 Markdown 图片引用，并保持图注
- 生成高反差 Swiss editorial 封面页（IKB 蓝网格、超大标题、研究电子书标识）
- 生成目录页（编号、引导线、报告式信息层级）
- 每个章节生成独立章节幕封（大字号章节名、网格背景、蓝色底栏）
- 正文使用研究报告式排版（窄版心、页眉标识、首段首字下沉、蓝色强调线）
- 图、表、截图以“视觉证据卡片”呈现，自动加入图注区与 `VISUAL NOTE` 标签
- 表格使用高反差表头、斑马纹行、细线分隔，适合论文数据阅读
- HTML 阅读版自动加入右侧浮动阅读目录，便于跳转封面、目录和章节；目录支持展开/收起，并在 PDF 导出时自动隐藏

### 设计标准（必须遵守）

生成 `translated.md` 时要为新版模板提供足够的结构信息：

1. **封面标题**
   - 第一行必须是一级标题 `# 书名：副标题` 或 `# 书名`。
   - 如果有副标题，用冒号分隔，模板会自动拆成主标题和副标题。

2. **章节结构**
   - 正文章节使用 `# 第1章 中文标题`、`# 第2章 中文标题`。
   - 阿拉伯数字和中文数字都支持；附录用 `# 附录：标题`。
   - 不要把大量正文直接堆在封面标题下，至少拆成明确章节。

3. **视觉材料**
   - 重要 Figure/Table/网页截图/模型架构图必须放入 `assets/` 并用 Markdown 图片引用：
     ```
     ![图 1：中文图题与关键结论](assets/figure1_xxx.png)
     ```
   - 图注不要只翻译标题，要补一句“这张图说明什么”，新版模板会把它设计成图表说明卡。
   - 宽表格、雷达图、论文网页截图、复杂图表优先保留原图；小型数据表可重建为 Markdown 表格。

4. **强调内容**
   - 需要形成“重点卡片”的段落可以写成单独一行加粗：
     ```
     **这一段会被排版成强调信息块。**
     ```
   - 引用或原文关键句使用 Markdown 引用 `> ...`。

5. **排版审美**
   - 不要写过长的章节标题，章节页标题控制在 8-18 个汉字最稳。
   - 不要在正文里裸露很长 URL；论文链接、网页来源优先做成截图或放到图注/参考信息中。
   - 图表文件名使用语义化命名，例如 `assets/figure1_comet_radar.png`、`assets/table3_ablation.png`。

6. **分页完整性**
   - 每页信息必须尽量完整，不能把同一张图、同一个表格、同一段图注、同一个重点块切到两页。
   - `build_pdf.py` 已对图片、图注、表格、引用块、重点块设置分页保护；生成 PDF 后仍要人工抽查关键页面。
   - 如果某张表格或图片本身超过一页高度，不要强行塞入正文：
     - 长表拆成多个小表，例如“表 3a / 表 3b / 表 3c”。
     - 宽表或复杂表优先裁剪成整页图片，并用单独图注解释关键结论。
     - 超大图表可以单独放在“图表页”或附录，不要夹在正文段落中间。
   - 标题后面至少要跟随一段正文或一张完整图表；不要让标题孤零零地留在页尾。
   - 图表插入位置应放在相关段落之后，而不是把正文逻辑硬切开。

如果脚本不在当前目录，从 skill 目录复制：
```
cp ~/.codex/skills/pdf-to-ebook/build_pdf.py ./
```

### 阶段 5：PDF 导出

1. 用 Python HTTP Server 托管 HTML：
   ```
   python3 -m http.server 8765 -d <output_dir> &
   ```
2. 用 `browser` 工具打开 `http://localhost:8765/<filename>.html`
3. 用 `browser action=pdf` 导出 PDF
4. 复制到目标位置（如桌面）

## 输出物

| 文件 | 说明 |
|------|------|
| `original.md` | 原始英文文本 |
| `translated.md` | 中文翻译 |
| `assets/` | 从原 PDF 提取或裁剪的图、表、截图等视觉资源 |
| `<书名>.html` | 精排 HTML |
| `<书名>.pdf` | 最终 PDF 电子书 |

## 注意事项

- 安装命令（brew/pip）超时 120s 直接跳过，不反复重试
- PDF 提取阶段遇到连续的图片化页面（如 InDesign 导出的后半部分），直接用优先级 4 或兜底方案
- 论文、报告、白皮书类 PDF 必须审查 Figure/Table，不能只交付纯文本译文
- 宽表格在 PDF 中可能很难重排，优先保留原图，同时给出中文摘要说明
- 翻译时保留英文术语，不强行翻译专业名词
- 默认不要生成朴素 Word 风格页面；HTML/PDF 应体现封面、目录、章节幕、图表卡片、重点块等设计层级
- 最终导出 PDF 后必须检查分页：图、表、图注、重点块、标题与其后正文不能出现明显割裂
- PDF 导出前确保关闭 Preview 进程（`osascript -e 'tell application "Preview" to quit'`）避免文件锁定
