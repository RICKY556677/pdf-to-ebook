# pdf-to-ebook

将英文 PDF 翻译为中文，并生成具有设计感的 HTML / PDF 中文电子书。

新版默认采用 **Swiss editorial / 研究报告电子杂志风格**：高反差封面、网格系统、章节幕封、图表证据卡片、可折叠 HTML 阅读目录，以及适合 A4 PDF 导出的稳定分页规则。目标不是把 PDF 简单转成 Word 式长文，而是把论文、报告、白皮书重新排成可阅读、可演示、可归档的中文电子书。

适合场景：

- 英文论文翻译成中文研究电子书
- 行业报告、白皮书、技术文档精排
- 带 Figure / Table / 架构图 / 网页截图的学术资料整理
- 需要同时交付 HTML 阅读版和 PDF 归档版的内容

## 概述

这是一个完整的自动化流程，能够：

1. **提取**英文 PDF 的文本内容（支持多种引擎，自动降级）
2. **审查并提取图表**（Figure、Table、图形、统计表、案例表、附录表）
3. **翻译**为中文（保留专业术语）
4. **排版**生成设计感更强的 A4 尺寸电子书（含高反差封面、目录、章节幕封、图表卡片）
5. **导出**最终 PDF 文件

## 核心特性

- **设计感排版**：默认 Swiss editorial 风格，视觉接近研究报告电子杂志。
- **HTML / PDF 双输出**：HTML 适合浏览器阅读，PDF 适合保存、分享、打印。
- **HTML 专用阅读目录**：右侧浮动目录支持展开 / 收起；导出 PDF 时自动隐藏。
- **统一响应式网格**：目录、章节幕封、正文、表格共享同一版心，拉宽浏览器也保持左边缘对齐。
- **图表优先工作流**：翻译前必须审查 Figure / Table / 图形 / 截图，不只处理纯文本。
- **视觉证据卡片**：图片、图表、网页截图会以卡片方式呈现，并带中文图注。
- **分页完整性保护**：尽量避免图片、图注、表格、引用块、重点块被切到两页。
- **Markdown 驱动**：用结构化 Markdown 控制章节、图表、重点块和引用。

## 工作流程

整个流程分为 5 个阶段：

### 阶段 1：PDF 文本提取

按优先级依次尝试，每次尝试后检查结果完整性：

| 优先级 | 方法 | 说明 |
|--------|------|------|
| 1 | 内置 `pdf` 工具 | 最快，首选 |
| 2 | `pdftotext`（poppler） | `brew install poppler` |
| 3 | PyMuPDF | `pip3 install PyMuPDF` |
| 4 | 图片 OCR | 针对图片化页面，通过 PyMuPDF 导出 PNG 后 OCR |
| 兜底 | 用户协助 | 请求用户手动导出文本或截图 |

每个方法最多尝试一次，安装命令超时 120 秒即跳过，不反复重试。

### 阶段 2：图表与图片审查

翻译前必须检查 PDF 中是否有需要保留的视觉信息：

- 搜索 `Figure`、`Fig.`、`Table`、`Chart`、`Diagram`、`Appendix` 等标记
- 建立图表清单：编号、标题、页码、正文引用位置
- 判断每个图表是重建为 Markdown 表格，还是裁剪为图片放入 `assets/`
- 对复杂图形、雷达图、统计图、模型架构图、宽表格，优先保留原图
- 在 `translated.md` 中用 Markdown 图片语法插入，并补中文图注

输出的电子书必须包含正文主要 Figure/Table 和附录关键表格；若无法提取，需要说明原因。

### 阶段 3：翻译

- 将提取的英文文本保存为 `original.md`
- 逐章翻译为中文，保存为 `translated.md`
- 保留英文术语（如 MVP、PMF、CAC、LTV、GTM）
- 保持原文格式：加粗、列表、引用、表格
- 超过 20 页时分批翻译

### 阶段 4：排版生成

使用 `build_pdf.py` 脚本生成 HTML 和 PDF，包含：

- **封面页**：IKB 蓝网格背景、超大标题、研究电子书标识
- **目录页**：编号、点状引导线、报告式信息层级
- **章节页**：独立章节幕封、大字号标题、网格背景、蓝色底栏
- **图表页/图注**：支持 Markdown 图片引用，以“视觉证据卡片”呈现
- **正文**：研究报告式正文版心、首字下沉、蓝色强调线、A4 尺寸
- **HTML 阅读目录**：右侧浮动目录用于网页阅读跳转，支持展开/收起，导出 PDF 时自动隐藏

### 阶段 5：PDF 导出

通过本地 HTTP Server 托管 HTML，使用浏览器工具导出为 PDF。

## 输出文件

| 文件 | 说明 |
|------|------|
| `original.md` | 原始英文文本 |
| `translated.md` | 中文翻译 |
| `assets/` | 提取或裁剪出的图、表、截图等视觉资源 |
| `<书名>.html` | 精排 HTML |
| `<书名>.pdf` | 最终 PDF 电子书 |

## 项目结构

```
pdf-to-ebook/
├── SKILL.md        # 工作流定义（触发条件、各阶段步骤）
├── build_pdf.py    # 排版脚本：Markdown → 精排 HTML
└── README.md       # 项目说明
```

## 使用方法

本项目作为 Codex / Claude Code Skill 使用，通过 `SKILL.md` 定义完整工作流。使用方式：

- 上传英文 PDF 并要求翻译成中文电子书
- 说"把这个 PDF 做成中文电子书"或"翻译这份 PDF 并排版"

也可以直接使用排版脚本把已经准备好的 Markdown 转成 HTML：

```bash
python3 build_pdf.py translated.md ebook.html
```

参数说明：

| 参数 | 说明 |
|------|------|
| `translated.md` | 已翻译并整理好的 Markdown 文件 |
| `ebook.html` | 输出 HTML 文件路径 |

图片和图表资源建议放在 Markdown 同级的 `assets/` 目录中。

## 排版效果

生成的电子书包含以下元素：

- A4 尺寸页面
- Swiss editorial 高反差封面
- 自动目录（点状引导线）
- 章节独立幕封
- 中文无衬线研究报告正文
- Figure/Table 原图嵌入、视觉卡片与中文图注
- 代码块、表格、引用块、重点信息块样式
- HTML 专用浮动目录，支持展开/收起，不进入最终 PDF
- 分页控制（封面/目录/章节页独立分页）

## HTML 与 PDF 的差异

| 能力 | HTML 阅读版 | PDF 导出版 |
|------|-------------|------------|
| 封面、目录、章节幕封 | 支持 | 支持 |
| 正文、图表卡片、表格 | 支持 | 支持 |
| 右侧浮动阅读目录 | 支持 | 自动隐藏 |
| 目录展开/收起 | 支持 | 不显示 |
| 响应式浏览器阅读 | 支持 | 固定 A4 版式 |
| 分页完整性保护 | 部分依赖浏览器滚动 | 用于导出 PDF |

## Markdown 写作建议

- 第一行使用 `# 书名：副标题`，模板会自动拆封面主副标题。
- 正文章节使用 `# 第1章 标题`，章节标题不要过长。
- 重要图表用 `![图 1：中文图题与关键结论](assets/figure1_xxx.png)` 插入。
- 需要突出显示的结论写成单独一行加粗，模板会排成重点信息块。
- 尽量不要裸露长 URL；论文网页、证据页面优先截图放入 `assets/`。

### 推荐 Markdown 结构

```markdown
# 论文中文标题：副标题

# 第1章 研究背景

这里是正文。

**这里是一条重点结论，会被排版成强调信息块。**

![图 1：这张图说明了模型在低频表达下的性能退化](assets/figure1.png)

| 方法 | 成本 | 适用场景 |
|------|------|----------|
| SLoW | 低 | Prompt 侧轻量干预 |
| 重训 tokenizer | 高 | 底层模型升级 |
```

## 分页完整性原则

- 图片、图注、表格、引用块、重点信息块默认会尽量保持在同一页内。
- 标题会尽量和后续内容保持在一起，避免页尾孤立标题。
- 长表如果超过一页高度，应拆成多个子表，或转为整页截图/附录图表。
- 宽图、复杂表、论文网页截图应优先作为完整视觉卡片插入，不要让正文把图注挤到下一页。
- 每次导出 PDF 后都要抽查分页，尤其是 Figure/Table 附近页面。
